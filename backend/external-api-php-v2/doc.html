TSC-NPS External API

API de consultation des paiements d'immatriculation. Permet aux partenaires bancaires de récupérer en temps réel les informations complètes d'un paiement à partir du numéro de plaque ou du numéro de transaction.

Base URL https://mpako.net/Backend/external-api-php-v2/api/index.php?route=
🔐
Authentification

Chaque requête protégée doit inclure deux headers HTTP. Ces credentials sont fournis lors de l'onboarding partenaire et gérés dans la table banques_partenaire.

Header 1
X-Bank-ID
Exemple
BANK-TSC-TEST-001
Header 2
X-API-Key
Type
Clé secrète opaque
Exemple de requête authentifiée
GET https://mpako.net/Backend/external-api-php-v2/api/index.php?route=paiement/plaque&plaque=SP222 Host: mpako.net X-Bank-ID: BANK-TSC-TEST-001 X-API-Key: tsc-api-key-test-2024-abcdef123456
⚠️ Sécurité : Ne partagez jamais votre clé API dans du code client ou des dépôts publics. En cas de compromission, contactez TSC-NPS immédiatement pour une rotation des credentials.

Health Check

Route publique, sans authentification. Vérifie que l'API et la base de données sont opérationnelles. Idéale pour le monitoring et les checks de disponibilité.

GET index.php?route=health Public · Sans auth ▶

curl -X GET \ "https://mpako.net/Backend/external-api-php-v2/api/index.php?route=health"

Recherche par Transaction

Retrouve un paiement via son numéro de référence de transaction unique. Utile pour le rapprochement bancaire après qu'un paiement a été effectué dans votre système.

GET index.php?route=paiement/transaction&ref={numero_transaction} 🔒 Auth requise ▶
Paramètre de requête
ParamètreTypeRequisDescription
ref string Oui Référence unique du paiement (ex : VIGN-1768822573481-3014)
cURL
curl -X GET \ "https://mpako.net/Backend/external-api-php-v2/api/index.php?route=paiement/transaction&ref=VIGN-1768822573481-3014" \ -H "X-Bank-ID: BANK-TSC-TEST-001" \ -H "X-API-Key: tsc-api-key-test-2024-abcdef123456"
Réponses
200 OK Structure identique à la réponse par plaque
{ "status": "success", "message": "Paiement récupéré avec succès", "data": { /* objet complet — voir endpoint par plaque */ } }
404 Not Found Référence de transaction inconnue
{ "status": "error", "message": "Paiement introuvable", "error": "Aucun paiement trouvé pour la référence 'VIGN-XXXX'" }
📦
Format des réponses

Toutes les réponses partagent la même enveloppe JSON. Les champs data et error sont mutuellement exclusifs selon le statut.

{ "status": "success" | "error", "message": "Message lisible par l'humain", "data": { /* présent UNIQUEMENT si status = "success" */ }, "error": "Détail technique" /* présent UNIQUEMENT si status = "error" */ }
Objets contenus dans data
ChampTypeDescription
site objectInformations du site TSC-NPS (id, nom_site, fournisseur)
assujetti objectPropriétaire du véhicule (nom, téléphone, adresse, nif, email)
engin objectDonnées du véhicule (plaque, marque, châssis, moteur…)
paiement objectDétails du paiement (montant, mode, transaction, date, statut)
repartition objectRépartition entre bénéficiaires (PROV, TSC…) avec montants
utilisateur objectAgent TSC-NPS ayant enregistré le paiement
⚠️
Codes d'erreur

L'API retourne des codes HTTP standards accompagnés d'un message et d'un détail explicite dans le corps JSON.

HTTPMessageCause
400Paramètre manquant Numéro de plaque ou de transaction absent ou vide dans l'URL
401Headers manquants X-Bank-ID ou X-API-Key absent du header de la requête
401Authentification échouéeBank ID ou API Key incorrect(e) — vérifier vos credentials
403Accès refusé Compte partenaire désactivé (actif = 0 en base)
403Compte suspendu Compte suspendu — la raison est incluse dans le champ error
403Credentials expirés La date d'expiration des credentials est dépassée
403IP non autorisée L'IP source n'est pas dans la whitelist configurée du partenaire
403Montant hors limites Montant du paiement en dehors des bornes min/max du partenaire
404Paiement introuvable Aucun paiement avec statut "completed" trouvé pour ce critère
404Route introuvable L'URL demandée n'existe pas dans l'API
500Erreur interne Erreur serveur inattendue — contacter le support TSC-NPS
🛡️
Limites & Sécurité

Chaque partenaire dispose de limites individuelles configurées en base. Les contrôles de sécurité sont appliqués à chaque requête, dans l'ordre indiqué.

Limite journalière
Configurée par partenaire
Limite mensuelle
Configurée par partenaire
Montant minimum
Défini par partenaire
Montant maximum
Défini par partenaire
Contrôle (ordre d'exécution)Description
1. Headers obligatoiresX-Bank-ID + X-API-Key présents et non vides
2. Credentials en base Vérification temps réel dans banques_partenaire
3. Compte actif Champ actif = 1 obligatoire
4. Non suspendu Champ suspendu = 0 obligatoire
5. Expiration Credentials vérifiés contre date_expiration
6. IP whitelist Si ip_autorisees est renseigné, l'IP source est contrôlée
7. Montant min/max Montant du paiement vérifié contre les bornes du partenaire
8. Anti injection SQL 100 % requêtes préparées PDO — aucune concaténation SQL
9. Audit trail Chaque connexion enregistrée dans connexions_bancaires
10. Fichiers protégés Accès direct à /classes/, .env, .sql bloqué par Nginx (404)
