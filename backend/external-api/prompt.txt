## ÉTAPE 1 — Installer Go sur le VPS

```bash
# Connexion SSH
ssh root@78.138.58.121

# Télécharger Go 1.22
wget https://go.dev/dl/go1.22.4.linux-amd64.tar.gz

# Installer (supprime l'ancienne version si elle existe)
rm -rf /usr/local/go
tar -C /usr/local -xzf go1.22.4.linux-amd64.tar.gz
rm go1.22.4.linux-amd64.tar.gz

# Ajouter Go au PATH (permanent)
echo 'export PATH=$PATH:/usr/local/go/bin' >> /etc/profile
source /etc/profile

# Vérifier
go version
# → go version go1.22.4 linux/amd64
```

## ÉTAPE 4 — Compiler le binaire Go

```bash
cd /opt/external-api

# Télécharger les dépendances
go mod tidy

# Compiler en binaire optimisé pour la production
# CGO_ENABLED=0 = binaire statique sans dépendances système
# -ldflags="-s -w" = binaire plus léger (supprime les symboles de debug)
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build \
  -ldflags="-s -w" \
  -o /opt/external-api/external-api-server \
  ./cmd/server/main.go

# Vérifier que le binaire est bien créé
ls -lh /opt/external-api/external-api-server
```

---

## ÉTAPE 5 — Créer le service Systemd (démarrage automatique)

```bash
cat > /etc/systemd/system/external-api.service << 'EOF'
[Unit]
Description=TSC-NPS External API (Go)
Documentation=https://78.138.58.121
After=network.target mysql.service
Requires=mysql.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/external-api

# Variables d'environnement DB
Environment="DB_HOST=localhost"
Environment="DB_PORT=3306"
Environment="DB_USER=c0willyam"
Environment="DB_PASSWORD=acmilan poli"
Environment="DB_NAME=c0mpako"
Environment="SERVER_PORT=:8090"

# Binaire à exécuter
ExecStart=/opt/external-api/external-api-server

# Redémarrage automatique en cas de crash
Restart=always
RestartSec=5s

# Sécurité : limiter les capacités du processus
NoNewPrivileges=true
PrivateTmp=true

# Logs
StandardOutput=journal
StandardError=journal
SyslogIdentifier=external-api

[Install]
WantedBy=multi-user.target
EOF

# Donner les droits sur le dossier à www-data
chown -R www-data:www-data /opt/external-api
chmod +x /opt/external-api/external-api-server

# Recharger systemd et activer le service
systemctl daemon-reload
systemctl enable external-api
systemctl start external-api

# Vérifier que ça tourne
systemctl status external-api
```

**Pour voir les logs en temps réel :**
```bash
journalctl -u external-api -f
```

## ÉTAPE 6 — Configurer Nginx comme reverse proxy

**Dans ISPConfig :** crée un site (ex: `api-tsc.tondomaine.cd`) ou utilise un sous-domaine.

Puis édite la configuration Nginx de ce site.
Si tu gères manuellement, crée le fichier :

```bash
cat > /etc/nginx/sites-available/external-api << 'EOF'
server {
    listen 80;
    server_name api-tsc.tondomaine.cd;  # ← Remplace par ton domaine ou IP

    # Logs spécifiques à cette API
    access_log /var/log/nginx/external-api-access.log;
    error_log  /var/log/nginx/external-api-error.log;

    # Sécurité : masquer la version Nginx
    server_tokens off;

    location / {
        # Proxy vers l'API Go sur le port 8090
        proxy_pass         http://127.0.0.1:8090;
        proxy_http_version 1.1;

        # Headers nécessaires pour que Go reçoive la vraie IP client
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;

        # Timeouts
        proxy_connect_timeout 10s;
        proxy_read_timeout    30s;

        # Transmettre les headers custom des partenaires
        proxy_pass_request_headers on;
    }

    # Health check accessible sans auth (pour monitoring)
    location /api/v1/health {
        proxy_pass http://127.0.0.1:8090;
        proxy_set_header Host $host;
        access_log off;
    }
}
EOF

# Activer le site
ln -s /etc/nginx/sites-available/external-api /etc/nginx/sites-enabled/

# Tester la config Nginx
nginx -t

# Recharger Nginx
systemctl reload nginx
```

> **Si tu utilises ISPConfig :** va dans Sites → Ton site → Directives Nginx supplémentaires et colle uniquement le bloc `location` :
> ```nginx
> location / {
>     proxy_pass http://127.0.0.1:8090;
>     proxy_set_header Host $host;
>     proxy_set_header X-Real-IP $remote_addr;
>     proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
>     proxy_pass_request_headers on;
> }
> ```
